Interface MutationObserver

Provides the ability to watch for changes being made to the DOM tree. It is designed as a replacement for the older Mutation Events feature which was part of the DOM3 Events specification.

observe

  • observe(target: Node, options?: MutationObserverInit): void
    • Defined in node_modules/typedoc/node_modules/typescript/lib/lib.dom.d.ts:10484

    Instructs the user agent to observe a given target (a node) and report any mutations based on the criteria given by options (an object). The options argument allows for setting mutation observation options via object members. These are the object members that can be used: childList Set to true if mutations to target's children are to be observed. attributes Set to true if mutations to target's attributes are to be observed. Can be omitted if attributeOldValue or attributeFilter is specified. characterData Set to true if mutations to target's data are to be observed. Can be omitted if characterDataOldValue is specified. subtree Set to true if mutations to not just target, but also target's descendants are to be observed. attributeOldValue Set to true if attributes is true or omitted and target's attribute value before the mutation needs to be recorded. characterDataOldValue Set to true if characterData is set to true or omitted and target's data before the mutation needs to be recorded. attributeFilter Set to a list of attribute local names (without namespace) if not all attribute mutations need to be observed and attributes is true or omitted.

    Parameters

    • target: Node
    • Optional options: MutationObserverInit

    Returns void
